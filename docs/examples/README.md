# Example Configurations

Ready-to-use configs for popular open source projects, generated by `gh velocity config preflight`.

## Usage

```bash
# Lead time for recent work
gh velocity lead-time -R <owner/repo> --config docs/examples/<file>.yml --since 30d

# Release analysis
gh velocity release <tag> --since <prev-tag> -R <owner/repo> --config docs/examples/<file>.yml -f json
```

## Generating Your Own

Use preflight to auto-detect a good starting config for any repo:

```bash
gh velocity config preflight -R owner/repo
gh velocity config preflight -R owner/repo --write  # saves to .gh-velocity.yml

# With a project board
gh velocity config preflight -R owner/repo --project-url https://github.com/users/me/projects/1
```

## E2E Testing

These configs are validated by the E2E test suite:

```bash
task e2e:configs
```

This builds the binary and runs each config against its target repo, verifying that valid JSON output with metrics is returned.

## Adding a New Example

1. Generate via preflight: `gh velocity config preflight -R <owner>/<repo> > docs/examples/<owner>-<repo>.yml`
2. Add an entry to `scripts/e2e-configs.sh` with `config|repo|tag|since_tag`
3. Run `task e2e:configs` to verify
